# Chapter 1 Specification

## 목적
Chapter 1의 목표는 핵심 동사로 실생활 영어를 더 빠르게 꺼내도록 돕는 것이다. 숫자 성공 횟수가 아니라, 표현을 뜻ㆍ문장ㆍ직접 출력 단계로 연결해 "영어 문장이 내 안에서 이어지는 느낌"을 키운다.

## 핵심 원칙
- 핵심 동사 중심 학습: `have`, `get`, `make`, `take`, `give`, `go`, `come`, `keep`, `feel`, `need`, `want`, `put`, `do`, `be` 등을 우선으로 한다.
- 명령문보다는 주어가 있는 일상 구어체를 우선한다.
- 표현은 단어 단위가 아니라 `동사 + 틀 + 명사` 연결 지도 관점으로 다룬다.
- 숫자 성공 카운트는 화면에 노출하지 않는다. 대신 연결도 시각화로 현재 상태를 알린다.
- `힌트 사다리`는 별도 메인 메뉴보다 `koen` 안의 보조 풀이로 제공하고, `문장 빌드 맵`은 사전 탐험 안의 코너로 제공한다.

## 표현 연결도 규칙
- 화면에는 `○—○—○` 형태의 3단계 연결도를 사용한다.
- 시각적으로는 `뜻 · 문장 · 말하기`로 표시한다.
- 세 단계는 각각 다음을 의미한다.
  1. 뜻 이해: 영어를 보고 뜻을 알거나 듣고 뜻을 연결한 경험
  2. 문장 조립: 덩어리 조립, 빈칸, 문장 빌드, 표현 틀 기반 문제 해결 경험
  3. 직접 꺼내기: 직접 영어 입력 정답 경험
- 표시 규칙
  - 아직 경험 없음: ○
  - 힌트 2~3단계 후 성공: ◐
  - 힌트 0~1단계로 성공: ●
  - 세 연결이 모두 강하게 이어짐: `●—●—● 🔗`
- 사용자에게는 처음 한 번만 안내 문구를 보여준다: “뜻을 알고, 문장을 만들고, 직접 꺼내면 표현이 연결돼요.”
- 카드와 상세 화면에는 긴 라벨 대신 시각적 연결도만 표시하고, 접근성을 위해 `aria-label`을 사용한다.

## 힌트 사용과 연결 강도 규칙
- 힌트 0~1단계 후 성공 시 strength 1
- 힌트 2~3단계 후 성공 시 strength 0.5
- 이미 강한 연결이 있을 때 낮은 점수로 다시 맞혀도 강도를 낮추지 않는다.
- `뜻은 전달됨` 상태는 `output` 연결을 채우지 않고 `timesCorrect`를 올리지 않는다.
- `뜻은 전달됨`은 `lastMeaningHitAt` 기록과 복습 우선순위 상승만 기록한다.

## Chapter 1 연결 기록 구조
- `etdQuestProgress.historyByExpressionId[expressionId]`에 저장한다.
- 각 표현의 연결 기록 구조:
  ```json
  {
    "lastAttemptedAt": null,
    "lastCorrectAt": null,
    "lastMeaningHitAt": null,
    "timesSeen": 0,
    "timesCorrect": 0,
    "lastHintLevel": 0,
    "reviewPriority": 0,
    "connections": {
      "recognition": { "strength": 0, "updatedAt": null },
      "assembly": { "strength": 0, "updatedAt": null },
      "output": { "strength": 0, "updatedAt": null }
    }
  }
  ```
- 매핑
  - `recognition`: `enToKo`, `listening` 등 의미 인식형 성공
  - `assembly`: `blank`, chunks 조립, 문장 빌드, `hint` 퀴즈 성공
  - `output`: 직접 영어 입력 정답 성공

## 약한 연결고리 복습 규칙
- `reviewPriority`는 0~10으로 제한한다.
- 가중치
  - 오답 또는 다시 연결 필요: +3
  - 뜻은 전달됨: +2
  - 힌트 3단계 사용 후 성공: +2
  - 아직 비어 있는 연결 조각 하나당: +1
  - 마지막 학습 후 7일 이상: +1
  - 힌트 없이 강하게 성공: -1
  - 세 연결이 모두 강함: -2
- 복습 문제 선정
  - `reviewPriority`가 높은 표현부터 우선
  - `recentExpressionIds`는 가능한 한 피한다
  - 같은 표현이 반복되지 않도록 후보를 섞는다
  - 뜻은 전달됐지만 목표 표현과 달랐던 문장도 후보에 포함한다
- 버튼 이름
  - 즉시 다시 풀기: “한 번 더 해보기”
  - 약한 표현 복습: “다시 연결하기”

## 화면 반영 위치
- 사전 탐험의 표현 카드
- 사전 탐험의 문장 빌드 맵 코너
- 표현 상세 화면
- 퀴즈 정답 결과 화면
- 약한 연결 복습 목록
- 오늘의 말 · 확장 고르기 홈 카드 (순서 없는 말하기 확장)

## 현재 범위와 제외 범위
- 포함
  - 연결도 시각화
  - 역사 기반 연결 강도 추적
  - 복습 우선순위 기반 review 목록
  - 한 번 맞힌 표현의 재출제
- 제외
  - 외부 라이브러리, npm, 서버, 로그인
  - PWA 서비스워커 수정
  - 음성 인식 정답 판정
  - 대규모 데이터 콘텐츠 추가

## 향후 확장 방향
- 음성 입력 정답을 `output` 연결에 포함
- 로그인 동기화를 통해 학습 이력을 여러 기기에서 유지
- 연결 지도 기반 표현 추천과 패턴별 약점 분석
- 연결 지도 확장: 동사 → 표현 틀 → 대표 범주 → 세부 명사/동사 → 자주 붙는 조합 → 예문
- 의문문 만들기 기능: 질문형 퀴즈와 패턴 기반 의문문 연습
- 실제 브라우저 마이크 입력 기능: 따라 말하기 및 음성 입력 정답 지원
