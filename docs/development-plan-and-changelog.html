English Thinking Dictionary Quest 개발 계획서 및 수정 내역

기준 버전: v2.1 / 작성일: 2026-07-06

1. 문서 목적

이 문서는 English Thinking Dictionary Quest의 현재 개발 방향과 지금까지 반영된 수정 내역을 한 번에 확인할 수 있도록 정리한 내부 문서입니다.

핵심 목표는 문장빌드맵 중심 콘텐츠 원본과 동사 카드 중심 탐색 메뉴를 분리하고, 게임모드가 그 원본을 소비하는 구조로 다시 정렬하는 것입니다.

2. 현재 개발 방향

  • 문장빌드맵은 콘텐츠 원본 화면입니다. 기본 엔진 → 생각틀 → 실생활 표현을 순서대로 보여줍니다.
  • 동사 카드는 기존 생각 사전 검색을 담는 탐색 메뉴입니다. 사용자는 동사를 먼저 보고, 그 뒤 표현과 문장빌드맵으로 이동합니다.
  • 게임모드는 문장빌드맵의 예문과 분류를 읽어 문제를 내는 소비자 화면입니다.
  • 생각틀은 2~3개로 제한하지 않고, 동사별로 5~6개까지 유연하게 둘 수 있습니다.
  • 실생활 표현은 생각틀별로 자주 쓰는 예문을 5개씩 보여주고, 예문 클릭 시 듣고 따라하기로 이어집니다.

3. 화면 역할 정리

화면 / 메뉴 역할
문장빌드맵 기본 엔진, 생각틀, 실생활 표현을 한 화면에서 보여주는 콘텐츠 원본
동사 카드 기존 생각 사전 검색 화면. 동사 탐색과 표현 탐색의 진입점
게임모드 듣고 따라하기, 한글→영어 조립, 영어→장면 고르기, 약한 연결 복습을 수행하는 퀘스트 화면
내 성장 진행률, 학습 이력, 약한 연결 복습 상태를 보여주는 성장 기록 화면

4. 지금까지 반영된 수정 내역

  • 상단 메뉴를 게임 모드 / 동사 카드 / 문장빌드맵 / 내 성장 구조로 재배치했습니다.
  • 사전 탐험 라벨을 정리하고, 동사 카드와 문장빌드맵으로 역할을 분리했습니다.
  • 문장빌드맵에 기본 엔진 / 생각틀 / 실생활 표현 구조를 적용했습니다.
  • 동사 카드에서 기존 생각 사전 검색의 핵심 동사를 카드 형태로 다시 볼 수 있도록 연결했습니다.
  • 게임모드의 듣기·따라 말하기·한영·영한·복습 흐름이 문장빌드맵의 새 분류와 맞도록 수정했습니다.
  • 서비스워커 캐시 버전을 갱신하고, 오래된 캐시를 정리하는 방향으로 반영했습니다.
  • 실생활 표현의 예문을 클릭하면 듣고 따라하기 흐름으로 이동하도록 연결했습니다.
  • 생각틀 선택 시 바로 아래 예문이 바뀌고, 첫 진입 시에는 랜덤한 생각틀이 선택되도록 했습니다.

5. 현재 작업 기준 정리

기본 엔진

동사의 중심 감각을 짧은 문장으로 정리합니다. 예: get은 얻다 · 받다 · 도착하다 · 이해하다 · 상태가 되다.

생각틀

동사가 실제로 붙는 자주 쓰는 덩어리입니다. 예: get + thing, get + place, get + adjective.

동사에 따라 생각틀 수는 다를 수 있고, 2~3개에 묶지 않아도 됩니다. 상황에 맞으면 5~6개까지 확장 가능합니다.

실생활 표현

생각틀별로 실제 원어민이 자주 쓰는 예문 5개를 보여줍니다. 예문은 중복 의미를 줄이고, 게임모드 출제와도 맞물리게 구성합니다.

6. 앞으로 확인할 것

  • 15개 핵심 동사 모두에서 기본 엔진, 생각틀, 실생활 표현이 균형 있게 분리되어 있는지 확인
  • 각 생각틀마다 예문 5개가 정확히 들어가는지 확인
  • 동사 카드와 문장빌드맵의 역할이 서로 중복되지 않는지 확인
  • 게임모드에서 듣고 따라하기가 실생활 표현 예문과 자연스럽게 이어지는지 확인
  • 브라우저 캐시와 서비스워커로 인해 이전 화면이 남지 않는지 확인
