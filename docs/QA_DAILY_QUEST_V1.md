# QA: 오늘의 7분 퀘스트 1차 E2E 검증

## 목적
홈 화면의 단일 데일리 퀘스트가 `quest:` 토큰 기준으로만 보이고, 별도의 퀘스트 카드가 더 이상 노출되지 않는지 실제 브라우저에서 검증한다.
보조 코너인 `힌트 사다리`와 `문장 빌드 맵`은 각각 `koen`과 사전 탐험 안에서만 노출되는지 함께 확인한다.

## 검증 환경
- 브라우저: Chrome/Safari 최신 버전
- 해상도:
  - 데스크톱: 1280px 이상
  - 모바일: 390px
- 서버: `python3 -m http.server 8080`

## E2E 테스트 케이스

### 1. 홈 진입 시 단일 데일리 퀘스트만 보임
**시나리오**:
1. localStorage 초기화
2. 홈 화면 로드

**예상 결과**:
- `오늘의 7분 퀘스트 시작` 버튼 표시
- `오늘의 최소 루프` 카드 표시
- `듣고 따라 말하기 1세트 → 영어 장면 고르기 2회 → 한글→영어 조립 2회 → 약한 연결 복습 1회`처럼 읽히고, `듣기`와 `따라 말하기`가 다른 메뉴처럼 중복 노출되지 않음
- 별도 퀘스트 카드나 옛 `0/3` 진행 표시는 노출되지 않음

**통과 조건**: ✅ / ❌

---

### 2. 시작 버튼이 별도 퀘스트 카드가 아니라 일반 플레이 흐름으로 이동함
**시나리오**:
1. 홈의 `오늘의 7분 퀘스트 시작` 클릭

**예상 결과**:
- 첫 퀘스트 스텝 화면으로 바로 이동
- `오늘의 최소 루프` 진행 상태는 홈 카드에서만 보임
- `힌트 사다리`와 `문장 빌드 맵`은 별도 카드로 열리지 않음
- `koen` 안의 보조 풀이와 사전 탐험 안의 빌드맵으로만 이어짐

**통과 조건**: ✅ / ❌

---

### 3. `daily.done`은 한 모드당 하루 한 번만 카운트됨
**시나리오**:
1. 오늘의 퀘스트 첫 스텝을 성공
2. 같은 스텝을 새로 다시 열어도 `quest:` 토큰은 중복 추가되지 않음

**예상 결과**:
- 첫 성공 시 `dailyCount`가 1 증가
- 같은 스텝 재성공 시 `dailyCount` 추가 증가 없음

**통과 조건**: ✅ / ❌

---

### 4. 7개 완료 신호가 합산되면 `7 / 7` 표시
**시나리오**:
1. 7개 퀘스트 스텝을 순서대로 완료
2. `hint`는 `koen` 안의 보조 풀이로, `build`는 사전 탐험 안의 문장 빌드 맵으로만 노출

**예상 결과**:
- `dailyCount`가 `7 / 7`
- `dailyPercent`가 `100%`
- 보조 코너는 별도 메인 메뉴처럼 노출되지 않음

**통과 조건**: ✅ / ❌

---

### 5. 날짜가 바뀌면 새 진행으로 시작됨
**시나리오**:
1. `daily.date`를 어제 날짜로 바꾸고 새로고침

**예상 결과**:
- `dailyCount`가 0으로 재계산
- `오늘의 최소 루프` 카드가 새 하루 기준으로 시작

**통과 조건**: ✅ / ❌

---

### 6. 일반 사전/지도/게임 퀴즈가 데일리 카운트에 중복 반영되지 않음
**시나리오**:
1. 사전 탐험 또는 연결 지도에서 같은 표현을 여러 번 풀이

**예상 결과**:
- 일반 퀴즈와 사전 탐험 성공은 `daily.done`에 직접 들어가지 않음
- 홈 데일리 진행 수가 불필요하게 늘어나지 않음

**통과 조건**: ✅ / ❌

---

### 7. 데스크톱(1280px) 및 모바일(390px) 가로 스크롤 없음
**시나리오**:
1. 브라우저 폭 1280px 설정
2. 홈 화면 확인
3. 브라우저 폭 390px 설정
4. 홈 화면 확인

**예상 결과**:
- 1280px: 홈 카드가 자연스럽게 정렬되고 가로 스크롤 없음
- 390px: 카드가 세로로 재배치되고 텍스트가 잘리지 않음

**통과 조건**: ✅ / ❌

## 결과 기록

| 테스트 | 통과 | 메모 |
|--------|------|------|
| 1. 단일 데일리 노출 | ⬜ | 별도 퀘스트 카드 미노출 확인 |
| 2. 시작 버튼 진입 | ⬜ | 시작 버튼이 별도 퀘스트 카드로 가지 않는지 확인 |
| 3. 중복 카운트 방지 | ⬜ | 같은 모드 재실행 시 `daily.done` 불변 확인 |
| 4. 7 / 7 도달 | ⬜ | 7개 모드 각각 1회 성공 후 `7 / 7` 확인 |
| 5. 날짜 변경 | ⬜ | 다음 날 기준으로 0부터 재시작 확인 |
| 6. 일반 퀴즈 격리 | ⬜ | 사전/지도 풀이가 중복 카운트로 이어지지 않는지 확인 |
| 7. 반응형 레이아웃 | ⬜ | 1280 / 390에서 가로 스크롤 없음 확인 |
