# QA: 오늘의 7분 퀘스트 1차 E2E 검증

## 목적
홈 화면의 단일 데일리 퀘스트가 `daily.done` 기준으로만 보이고, 별도의 퀘스트 카드가 더 이상 노출되지 않는지 실제 브라우저에서 검증한다.

## 검증 환경
- 브라우저: Chrome/Safari 최신 버전
- 해상도:
  - 데스크톱: 1280px 이상
  - 모바일: 390px
- 서버: `python3 -m http.server 8080`

## E2E 테스트 케이스

### 1. 홈 진입 시 단일 데일리 퀘스트만 보임
**시나리오**:
1. localStorage 초기화
2. 홈 화면 로드

**예상 결과**:
- `오늘의 7분 퀘스트 시작` 버튼 표시
- `오늘의 최소 루프` 카드 표시
- 별도 퀘스트 카드나 `0/3` 진행 표시는 노출되지 않음

**통과 조건**: ✅ / ❌

---

### 2. 시작 버튼이 3문장 퀘스트가 아니라 일반 플레이 흐름으로 이동함
**시나리오**:
1. 홈의 `오늘의 7분 퀘스트 시작` 클릭

**예상 결과**:
- `게임 모드` 화면으로 이동
- 3문장 전용 퀘스트 카드가 열리지 않음

**통과 조건**: ✅ / ❌

---

### 3. `daily.done`은 한 모드당 하루 한 번만 카운트됨
**시나리오**:
1. `listen` 모드에서 정답 1회
2. 같은 날 `listen` 모드를 다시 정답

**예상 결과**:
- 첫 성공 시 `dailyCount`가 1 증가
- 같은 모드 재성공 시 `dailyCount` 추가 증가 없음

**통과 조건**: ✅ / ❌

---

### 4. 7개 모드 완료 시 `7 / 7` 표시
**시나리오**:
1. `listen`, `speak`, `koen`, `enko`, `hint`, `build`, `review`를 각각 한 번씩 성공

**예상 결과**:
- `dailyCount`가 `7 / 7`
- `dailyPercent`가 `100%`

**통과 조건**: ✅ / ❌

---

### 5. 날짜가 바뀌면 새 진행으로 시작됨
**시나리오**:
1. `daily.date`를 어제 날짜로 바꾸고 새로고침

**예상 결과**:
- `dailyCount`가 0으로 재계산
- `오늘의 최소 루프` 카드가 새 하루 기준으로 시작

**통과 조건**: ✅ / ❌

---

### 6. 일반 사전/지도/게임 퀴즈가 데일리 카운트에 중복 반영되지 않음
**시나리오**:
1. 사전 탐험 또는 연결 지도에서 같은 표현을 여러 번 풀이

**예상 결과**:
- 같은 모드 ID가 이미 들어가 있으면 `daily.done` 길이는 변하지 않음
- 홈 데일리 진행 수가 불필요하게 늘어나지 않음

**통과 조건**: ✅ / ❌

---

### 7. 데스크톱(1280px) 및 모바일(390px) 가로 스크롤 없음
**시나리오**:
1. 브라우저 폭 1280px 설정
2. 홈 화면 확인
3. 브라우저 폭 390px 설정
4. 홈 화면 확인

**예상 결과**:
- 1280px: 홈 카드가 자연스럽게 정렬되고 가로 스크롤 없음
- 390px: 카드가 세로로 재배치되고 텍스트가 잘리지 않음

**통과 조건**: ✅ / ❌

## 결과 기록

| 테스트 | 통과 | 메모 |
|--------|------|------|
| 1. 단일 데일리 노출 | ⬜ | 별도 퀘스트 카드 미노출 확인 |
| 2. 시작 버튼 진입 | ⬜ | 시작 버튼이 3문장 퀘스트로 가지 않는지 확인 |
| 3. 중복 카운트 방지 | ⬜ | 같은 모드 재실행 시 `daily.done` 불변 확인 |
| 4. 7 / 7 도달 | ⬜ | 7개 모드 각각 1회 성공 후 `7 / 7` 확인 |
| 5. 날짜 변경 | ⬜ | 다음 날 기준으로 0부터 재시작 확인 |
| 6. 일반 퀴즈 격리 | ⬜ | 사전/지도 풀이가 중복 카운트로 이어지지 않는지 확인 |
| 7. 반응형 레이아웃 | ⬜ | 1280 / 390에서 가로 스크롤 없음 확인 |
